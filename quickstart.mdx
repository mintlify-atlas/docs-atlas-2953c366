## Prerequisites

Before you begin, make sure the following tools are installed on your machine:

| Tool | Purpose | Minimum version |
|------|---------|-----------------|
| [Docker Desktop](https://www.docker.com/products/docker-desktop/) | Runs the SQL Server container | Any current release |
| [.NET 8 SDK](https://dotnet.microsoft.com/en-us/download/dotnet/8.0) | Builds and runs the ASP.NET Core backend | 8.0 |
| [Node.js](https://nodejs.org/) | Runs the React frontend | 18+ |

<Tip>Run `dotnet --version` and `node --version` in a terminal to confirm your installations before continuing.</Tip>

## Setup

<Tabs>
  <Tab title="Backend">

<Steps>
  <Step title="Navigate to the Backend folder">
    ```bash
    cd MeetPoint/Backend
    ```
  </Step>

  <Step title="Create appsettings.json">
    Create a new file called `appsettings.json` in the `Backend/` directory and paste the following content.

    <Warning>The SQL Server password in `appsettings.json` must match `MSSQL_SA_PASSWORD` in `docker-compose.yml`. Both default to `YourStrong@Password123` — change them together if you use a custom password.</Warning>

    ```json appsettings.json
    {
      "ConnectionStrings": {
        "DefaultConnection": "Server=localhost,1433;Database=MeetPoint;User Id=sa;Password=YourStrong@Password123;Trusted_Connection=false;TrustServerCertificate=true;Encrypt=true;"
      },
      "AllowURLS": [
        "http://localhost:5173"
      ],
      "PageSize": 8,
      "JWT": {
        "ValidAudience": "http://localhost:5173",
        "ValidIssuer": "https://localhost:7191",
        "Secret": "Esta-es-una-clave-super-secreta-y-segura-para-JWT-con-al-menos-32-caracteres",
        "Expires": "120",
        "RefreshTokenExpire": "240"
      },
      "Logging": {
        "LogLevel": {
          "Default": "Information",
          "Microsoft.AspNetCore": "Warning"
        }
      }
    }
    ```

    <Note>Port numbers may vary depending on your environment. Update `AllowURLS`, `ValidAudience`, and `ValidIssuer` if your frontend or backend run on different ports.</Note>
  </Step>

  <Step title="Start the SQL Server container">
    The `docker-compose.yml` spins up a SQL Server 2022 container named `meetpoint_sqlserver` on port `1433`.

    ```bash
    docker-compose up -d
    ```

    The container uses a health check and retries up to 10 times. Wait a few seconds for it to become healthy before running migrations.

    <Tip>Run `docker ps` to confirm the container is running and healthy.</Tip>
  </Step>

  <Step title="Install Entity Framework tools">
    Skip this step if you already have `dotnet-ef` installed globally.

    ```bash
    dotnet tool install --global dotnet-ef
    ```
  </Step>

  <Step title="Restore dependencies">
    ```bash
    dotnet restore
    ```
  </Step>

  <Step title="Run database migrations">
    Create and apply the initial migration to set up the database schema:

    ```bash
    dotnet ef migrations add InitDatabase
    dotnet ef database update
    ```

    <Note>On first run, the application also seeds initial data — including the test admin and user accounts — automatically via the `MeetPointSeeder`.</Note>
  </Step>

  <Step title="Start the backend">
    ```bash
    dotnet run
    ```

    The API will be available at `http://localhost:5048`. Swagger UI is accessible at `http://localhost:5048/swagger` while running in development mode.
  </Step>
</Steps>

  </Tab>
  <Tab title="Frontend">

<Steps>
  <Step title="Navigate to the Frontend folder">
    ```bash
    cd MeetPoint/Frontend
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    npm install
    ```

    This installs React 18, Vite, Tailwind CSS, Zustand, Formik, Axios, and all other dependencies listed in `package.json`.
  </Step>

  <Step title="Create the .env file">
    Create a `.env` file in the `Frontend/` directory with the backend API URL:

    ```bash .env
    # URL of the backend API
    VITE_API_URL=http://localhost:5048/api
    ```

    <Warning>The `VITE_API_URL` must match the port your backend is running on. If the backend uses a different port, update this value accordingly.</Warning>
  </Step>

  <Step title="Start the development server">
    ```bash
    npm run dev
    ```

    The frontend will be available at `http://localhost:5173`. The Vite dev server supports hot module replacement — changes appear instantly without a full reload.
  </Step>
</Steps>

  </Tab>
</Tabs>

## What to try first

Once both services are running, here are a few things to explore:

<Steps>
  <Step title="Browse the home page">
    Open `http://localhost:5173` to see the event feed. You can browse public events without logging in.
  </Step>

  <Step title="Log in as a user">
    Sign in with `user@me.com` / `Temporal01*` to access the full user experience — create an event, register attendance, leave a comment, and rate an event.
  </Step>

  <Step title="Create an event">
    After logging in, navigate to the event creation form. Fill in a title, description, category, date, and location, then submit to see it appear in the feed.
  </Step>

  <Step title="Explore the admin dashboard">
    Sign in with `admin@me.com` / `Temporal01*` to access the admin dashboard. From there you can manage users, events, categories, attendances, comments, and reports.
  </Step>
</Steps>
